    GitHub Copilot Prompting Workshop - Exercises¶

    This workshop teaches GitHub Copilot prompting mastery through 15 progressive exercises. You’ll evolve a simple Python calculator from ~15 lines to 350+ lines using ONLY prompting‚Äîno manual coding!

    Workshop Structure¶

    Duration: 4 hours
    Validation: Run pytest tests/test_exN.py after each exercise
    Core Principle: ALL code additions done via Copilot prompting

    Exercise 2: Chat Ask - Quick Enhancements (10 min)¶

    Copilot Mode: GitHub Copilot Chat (Ask mode)
    Starting Point: Your Ex1 code with 4 operations

    Your Task: Use Chat to ASK Copilot for help: - Add modulo(x, y) operation - Add power(x, y) operation (exponentiation)

    Example Prompts:

    ‚úÖ Validation & Progress Tracking¶

    After Each Exercise:¶

    1. Run the test suite: 

      pytest tests/test_ex1.py -v  # Replace with your exercise number

    2. Check for passing tests:

    3. All tests should pass (green)
    4. If tests fail, read the error message

    5. Use Copilot Chat: “Why is test_X failing?”

    6. Iterate if needed:

    7. Refine your prompts based on test failures
    8. Ask Copilot to fix specific issues

    9. Re-run tests until all pass

    10. Move to next exercise once tests pass

    Troubleshooting Tips:¶

    If inline suggestions aren’t working well: - Make your comments more specific - Write function signatures first - Try Chat Ask mode instead

    If Chat responses are too vague: - Add more context about your code - Be more specific about requirements - Switch to Plan mode for complex tasks

    If Agent goes in wrong direction: - Interrupt and redirect: “Stop, try a different approach” - Provide more constraints upfront - Break task into smaller steps

    If tests are failing: - Use Chat: “Why is [test_name] failing?” - Check test file to understand requirements - Iterate your prompts until tests pass
